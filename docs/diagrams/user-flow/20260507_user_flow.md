# Master User Flow — Magic Nugger API

Connects all API routes into end-to-end user journeys.

---

## 1. Complete Player Journey

```mermaid
flowchart TD
    START([New User]) --> ONBOARD

    subgraph ONBOARD["Onboarding — /auth"]
        direction TB
        REG["POST /auth/register\n{username, email, password}"] --> SESSION[(Authenticated Session)]
        LOGIN["POST /auth/login\n{username, password}"] --> SESSION
        OAUTH["GET /auth/oauth/google\n→ callback → /levels"] --> SESSION
    end

    SESSION --> DISCOVERY

    subgraph DISCOVERY["Level Discovery — /levels"]
        direction TB
        GET_LEVELS["GET /levels\nList all active levels\nordered by order_index"] --> PICK_LEVEL{Pick a level}
        PICK_LEVEL --> GET_LEVEL["GET /levels/:id\nGet level details"]
    end

    GET_LEVEL --> GAME_LOOP

    subgraph GAME_LOOP["Gameplay Loop — /game"]
        direction TB
        START_GAME["POST /game\n{level_id}"] --> CHECK_SESSION{Existing session?}
        CHECK_SESSION -->|within resume window| RESUME[Resume session 200]
        CHECK_SESSION -->|expired or none| NEW_SESSION[New session 201]
        RESUME --> ANSWER
        NEW_SESSION --> ANSWER
        ANSWER["POST /game/:id/answer\n{is_correct, time_taken_ms}"] --> MORE{More questions?}
        MORE -->|yes| ANSWER
        MORE -->|no, success| END_GAME["POST /game/:id/end"]
        MORE -->|no, fail| FAIL_GAME["POST /game/:id/fail"]
        MORE -->|quit| ABANDON["POST /game/:id/abandon"]
    end

    END_GAME --> POST_GAME
    FAIL_GAME --> POST_GAME
    ABANDON --> BACK_TO_LEVELS

    subgraph POST_GAME["Post-Game — ELO + Progression"]
        direction TB
        ELO_UPDATE["ELO updated\ncurrent_elo = GREATEST 0 currentElo + delta\nreason = session_completed or session_failed"]
        LEVEL_UP["highest_level_unlocked updated\nif completed and next level exists"]
        ELO_HIST["elo_history record appended"]
        CACHE_BUST["leaderboard cache invalidated\nscope=global + scope=level"]
    end

    POST_GAME --> LEADERBOARDS

    subgraph LEADERBOARDS["Leaderboards — /leaderboard"]
        direction LR
        LB_GLOBAL["GET /leaderboard/global\ncursor by current_elo"]
        LB_LEVEL["GET /leaderboard/levels/:id\ncursor by best_score\nfilter by period"]
        LB_CLASS["GET /leaderboard/classrooms/:id\ncursor by classroom_elo\nfilter by period"]
    end

    LEADERBOARDS --> SOCIAL

    subgraph SOCIAL["Classrooms — /classrooms"]
        direction TB
        CREATE_CLASS["POST /classrooms\n{name, visibility, starting_elo, elo_cap}\nreturns invite_code"]
        JOIN_CLASS["POST /classrooms/join\n{invite_code}"]
        VIEW_CLASS["GET /classrooms/:id"]
        UPDATE_CLASS["PATCH /classrooms/:id\nteacher only"]
        LEAVE_CLASS["DELETE /classrooms/:id/leave"]
    end

    SOCIAL --> PROFILE

    subgraph PROFILE["Player Profile — /players"]
        direction TB
        VIEW_PLAYER["GET /players/:id\nid, username, display_name\ncurrent_elo, highest_level_unlocked, avatar_url"]
        UPDATE_PLAYER["PATCH /players\n{display_name, username, avatar_url}"]
    end

    PROFILE --> GET_LEVELS
    LEADERBOARDS --> GET_LEVELS
    BACK_TO_LEVELS --> GET_LEVELS

    SESSION --> LOGOUT["POST /auth/logout"]
    SESSION --> ME["GET /auth/me"]
```

## 3. Session State Machine

```mermaid
stateDiagram-v2
    direction LR

    [*] --> in_progress : POST /game — start new session
    in_progress --> in_progress : POST /game — resume within window
    in_progress --> abandoned : POST /game/:id/abandon\nor POST /game — new start after expiry
    in_progress --> completed : POST /game/:id/end\nELO delta applied — level may unlock
    in_progress --> failed : POST /game/:id/fail\nELO delta discarded — level unchanged

    completed --> [*] : leaderboard cache invalidated\nelo_history appended
    failed --> [*] : leaderboard cache invalidated\nelo_history appended
    abandoned --> [*] : ELO reconciled on next start if delta != 0
```

## 4. ELO Flow

```mermaid
flowchart LR
    PLAYER["Player\ncurrent_elo"] --> SESSION_START["Session Start\nelo_before = current_elo"]
    SESSION_START --> ANSWERS["Answer Loop\nelo_delta ± per answer"]

    ANSWERS --> COMPLETED{Outcome}
    COMPLETED -->|completed| ELO_APPLY["finalElo = GREATEST 0 current_elo + delta\nreason: session_completed"]
    COMPLETED -->|failed| ELO_KEEP["finalElo = current_elo unchanged\nreason: session_failed"]
    COMPLETED -->|abandoned| ELO_RECONCILE["If delta != 0:\ncurrent_elo updated on next start\nreason: session_abandoned"]

    ELO_APPLY --> ELO_HIST["INSERT elo_history\n{player_id, session_id, elo_before, elo_after, delta, reason}"]
    ELO_KEEP --> ELO_HIST
    ELO_RECONCILE --> ELO_HIST

    ELO_HIST --> LB_INVALIDATE["Leaderboard cache invalidated\nscope=global + scope=level"]
```

## 5. Logging Coverage Map

| Trigger | Event | Level |
|---|---|---|
| Register | *(none — implicit via session)* | — |
| Login (local) | `auth:login` | info |
| Login (OAuth) | `auth:oauth_login` | info |
| Logout | `auth:logout` | info |
| Unauthorized access | `auth:unauthorized` | warning |
| Session started | `session:started` | info |
| Session resumed | `session:resumed` | info |
| Session completed | `session:ended` | info |
| Session failed | `session:failed` | info |
| Session abandoned | `session:abandoned` | info |
| ELO updated (game) | `elo:updated` | info |
| ELO adjusted (admin) | `admin:elo_adjusted` + `elo:admin_adjusted` | info |
| Role changed (admin) | `admin:role_changed` | info |
| Players viewed (admin) | `admin:player_viewed` | info |
| Sessions viewed (admin) | `admin:sessions_viewed` | info |
| Cache hit | `cache:hit` | info |
| Cache miss | `cache:miss` | info |
| Cache cleared | `cache:pruned` | info |
| Schema validation fail | `error:schema-validation` | error |
| Client-side event | configurable `LogEvent` | configurable `LogLevel` |

All log events are written to `audit.log_events` (partitioned, indexed by event + created_at + user_id).
